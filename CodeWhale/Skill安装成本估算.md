# Skill 安装成本估算

## 文件大小

| 文件 | 大小 |
|---|---|
| CodeConstitution/SKILL.md | 18,423 bytes |
| CodeConstitution/companion/checklist.md | 3,080 bytes |

| frontend-design/SKILL.md | ~12,000 bytes |
| karpathy-guidelines/SKILL.md | ~4,500 bytes |
| **合计** | **~50 KB** |

## Token 估算

| 维度 | 数值 |
|---|---|
| 估计 Token 数 | **~12,000 ~ 15,000 tokens** |
| CodeWhale 上下文上限 | 1,000,000 tokens |
| 占用比例 | **~1.2% ~ 1.5%** |
| 系统 prompt 现有大小 | ~5,000 tokens（估算） |
| 四个 Skill 加载后总 prompt | ~17,000 ~ 20,000 tokens |

## 影响分析

| 项目 | 安装前 | 安装后 | 变化 |
|---|---|---|---|
| 基础 prompt 大小 | ~5,000 tokens | ~17,000~20,000 tokens | +3~4 倍 |
| 占总上下文比例 | 0.5% | **~2%** | 可控 |
| **首轮响应速度** | 基准 | 慢 **18~28%** | 需多读 15K tokens |
| **后续对话** | 基准 | 几乎无差异 | 后续 token 增量相同 |
| **输出代码质量** | 基准 | **明显提升** | 按规范自动满足边界条件、日志、注释 |
| **成本（输入）** | 基准 | **+~$0.0025/次** | 以 $0.14/M 输入计，约 2.5 厘/次 |

## 结论

- **响应速度**：首次请求因需加载规则，会慢 18~28%（约 1~3 秒），后续对话基本无感知
- **上下文占用**：仅占 1M 窗口的 ~2%，完全可忽略
- **成本增量**：每次对话约多花 2.5 厘人民币，几乎可以不计
- **收益**：AI 输出的代码会自动满足边界条件防御、结构化日志、无歧义注释、前端设计规范等 250+ 条规范
